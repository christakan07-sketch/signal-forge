# Intake Prototype

## Purpose
This is the first runnable concept for turning a headline into structured research artifacts.

## Prototype flow
1. headline input
2. event normalization
3. case creation
4. taxonomy classification
5. study suggestion
6. evidence stub creation

## Goal
Prove that a single incoming event can become:
- a normalized event record
- a case-library entry
- a suggested study path
- an evidence stub for later observation

## Important note
This prototype is intentionally lightweight and document-first.
The point is to validate workflow before building automation.
