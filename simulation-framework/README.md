# Simulation Framework

## Purpose
This framework turns case-library events into repeatable research studies.

## Main study types
- persistence study
- follow-through study
- false-positive study
- cross-asset confirmation study

## Suggested workflow
1. choose a case from `case-library/cases/`
2. classify it using taxonomy
3. pick the study template that matches the event profile
4. define windows, assets, and failure criteria
5. record results in a study file

## Important rule
Simulation is for evidence gathering, not for pretending an edge already exists.
