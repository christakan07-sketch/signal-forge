# Starter Stack

## Goal
Use the lightest stack that still produces useful research output.

## Recommended prototype stack
### Event intake
- manual headline entry
- RSS feeds
- Reuters-grounded search
- official regulator / central bank / exchange pages

### Market data
- public OHLCV APIs for crypto
- lightweight ETF / proxy snapshots for sectors and commodities
- basic FX / index snapshots for macro comparison

### Internal processing
- case library
- taxonomy classification
- study templates
- evidence sheets

## Recommended research stack later
- curated news API
- unified OHLCV collector
- benchmark registry
- event linkage database
- automated follow-up headline tracking

## What not to do too early
- do not start with expensive real-time feeds before the taxonomy and case workflow prove useful
- do not overbuild execution logic before evidence collection works
