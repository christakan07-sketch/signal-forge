# Event Dimensions

This file defines additional dimensions beyond `event_type` so cases can be compared more intelligently.

## 1) Event family
Broad family of event:
- supply_shock
- demand_or_flow
- policy_signal
- regulatory_regime
- operational_incident
- macro_data
- company_event

## 2) First-order impact path
The most immediate transmission channel:
- physical_supply
- funding_or_liquidity
- regulatory_risk_premium
- policy_rate_expectation
- sentiment_repricing
- earnings_or_margin_pressure

## 3) Second-order impact path
Common follow-on channels:
- inflation_expectations
- sector_rotation
- valuation_rerating
- volatility_expansion
- spread_widening
- narrative_reinforcement

## 4) Persistence profile
How these events often behave:
- one_headline_spike
- short_follow_through
- regime_shift_candidate
- slow_burn_repricing
- highly_reversible

## 5) Typical validation method
What usually validates the case best:
- price_reaction_window
- relative_performance
- volatility_response
- rates_and_fx_reaction
- follow_up_policy_confirmation
- repeated_headline_pattern
- cross_asset_confirmation

## 6) Typical failure mode
How the idea often breaks:
- already_priced
- weak_causal_link
- symbolic_policy_only
- fast_reversal
- broad_beta_overwhelms_signal
- attribution_is_unclear
